Hexo搭建个人博客

记性不好，博客还是要写起来，方便我寄几查看，2333

下载安装

  1. 下载node.js： 下载地址nodes.org
    查看安装版本：
    ‘’’shell
    node -v
    npm -v
    ‘’’
  2. 安装镜像源（加快速度）
    切换到管理员模式
    ‘’’shell
    sudo su
    ‘’’
    安装淘宝镜像源
    ‘’’shell
    npm install -g cnpm —registry=https://registry.npm.taobao.org
    ‘’’
  3. 安装hexo博客框架
    ‘’’shell
    cnpm install -g hexo-cli
    ‘’’
    查看版本
    ‘’’shell
    hexo -v
    ‘’’
    安装下载准备工作完成后就可以退出管理员模式了。

搭建博客

  1. 切换到你想要的路径建立文件夹
    ‘’’shell
    mkdir myblog
    ‘’’
  2. 进入blog目录
    ‘’’shell
    cd myblog/
    ‘’’
  3. 初始化一个博客
    ‘’’shell
    hexo init
    ‘’’
  4. 启动博客
    ‘’’shell
    hexo s // 先在本地看一下博客
    ‘’’
  5. 新建文章
    ‘’’shell
    // 新建
    hexo n “my_first_blog”
    // 进入新建文章所在的目录下
    ls
    cd source/_posts/
    ls -l
    // 修改添加内容
    vim my_first_blog.m
    // 保存退出 :wq
    ‘’’
  6. 退回到myblog目录下
    ‘’’shell
    cd ../..
    ‘’’
  7. 本地生成启动
    ‘’’shell
    // 清理
    hexo clean
    // 生成
    hexo g
    // 启动
    hexo s
    ‘’’

    部署到远端

  8. github建仓（注意仓库名字必须是:用户名.giyhub.io）
  9. 安装部署插件
    ‘’’shell
    cnpm install —save hexo-deployer-git
    ‘’’
  10. 设置config
    ‘’’
    vim _config.yml
    ‘’’
  11. 部署到远端
    ‘’’shell
    hexo d
    ‘’’

切换主题风格：

‘’’shell
vim _config.yml
‘’’
Theme:ilia
Hexo clean
Hexo g
Hexo s
Hexo d

[SQL]

DQL（Data Query Language）数据查询语句

1.基础查询

语法：
select 查询列表
from 表名;
特点：
查询列表可以是：表中的字段、常量、表达式、函数
查询结果是一个虚拟的表格
注意：
在进行查询操作之前要指定所有的库：use + 库名；
查询：
查询表中的单个字段：select last_name from employees;
查询表中的多个字段：select last_name, salary, email from employees;
查询表中的所有字段：select * from employees;
查询常量值：select 100; select ‘john’;
查询表达式：select 100*98；
格式化：
按F12进行格式化
区分关键字：
着重号`用来区分是否是关键字或者字段
语句执行：
选中语句进行执行或F9
起别名：
便于理解
如果查询的字段有重名的情况，使用别名可以区分开来
方式1：select 100%98 as 结果;
select last_name as 姓, first_name as 名 from employees;
方式2：select last_name 姓, first_name 名 from employees;
如果别名有特殊符号要加双引号：elect salary as “out put” from employees;
去重：
查询员工表中涉及到的所有部门编号：select distinct department_id from employees;
+号的作用：
两个操作数为数值型，则做加法运算
只要其中一方为字符型，试图将字符型数值转换成数值型，如果转换成功，则继续做加法运算；如果转换失败，则将字符型数值转换成0
只要其中一方为null，则结果肯定为null
使用concat连接字符串：
查询员工的名和姓连接成一个字段，并显示为姓名：select concat(last_name,first_name) as 姓名 from employees;
ifnull和isnull：
ifnull函数检测是否为null，如果为null，则返回指定的值，否则返回原本的值；
isnull函数判断某字段或表达式是否为null，如果是，则返回1，否则返回0；

2.条件查询

语法：
select 查询列表
from 表名
where 筛选条件；
分类：
按条件表达式筛选：> < = != <> >= <=
按逻辑表达式筛选：&& || ! and or not
模糊查询：
(1) like (搭配通配符：%任意多个字符；_任意单个字符)
(2) between and
(3) 判断null的值：is null
安全等于：<=>
(4) in

3.排序查询

语法：
select 查询列表
from 表名
【where 筛选条件】
order by 排序列表 【asc｜desc】
说明：
asc代表的是升序，desc代表的是降序，如果不写，默认是升序
order by子句中可以支持单个字段、多个字段、表达式、函数、别名
order by子句一般是放在查询语句的最后面，但limit子句除外

【C++】STL容器特性总结

STL容器特性总结

1.Sequence Containers:

维持顺序的容器

vector:
    动态数组；
    O(1)随机读取，O(1)尾部增删；
    可作stack用。

list :
    双向链表；
    可做stack和queue用。

deque:
    双端队列；
    O(1)随机读取，O(1)头部尾部增删。

2.Container Adaptors:

基于其他容器实现的数据结构

stack:
    栈，后入先出；
    常用于深度优先、字符串匹配、单调栈问题；
    默认基于deque。

queue:
    队列，先入先出；
    常用于广度优先搜索；
    默认基于deque。

priority_queue:
    最大值先出的数据结构；
    用O(nlogn)的时间排序数组，O(nlogn)的时间插入任意值，O(1)的时间获得最大值, O(logn)的时间删除最大值；
    常用于维护数据结构并快速获取最大最小；
    默认基于vector。

3.Associative Containers:

实现了排好序的数据结构

set:
    有序集合，元素不可重复；
    O(nlogn)排序数组，插入删除查找任意值，获得最大最小值。

multimap:
    支持重复元素的map。

4.Unordered Associative Containers:

对每个Associative Containers实现了哈希版本

unordered_set:
    哈希集合；
    O(1)插入、查找、删除元素；
    常用于快速查询一个元素是否在这个容器内。

unordered_multiset:
    支持重复元素的unordered_set。

unordered_map:
    哈希映射或哈希表；
    在unordered_set基础上加上映射关系，可以对每一个元素key存一个值value。

unordered_multimap:
    支持重复元素的unordered_map。

linux基本命令

一. 登录注销重启

root——登录；
logout——登出； （Ctrl+D）
reboot——重启；
poweroff——关闭Linux系统。

二. 操作命令

列表命令

  1. 在命令行中输入“ls”——查看当前目录下所有文件及目录；
  2. 在命令行中输入“ls -l”（也可以输入“ll”）——当前目录下所有文件的详细信息；
  3. 在命令行中输入“ls install.log”——单独查看当前目录下名为install.log的文件，一般用来确认当前目录是否存在该文件；
  4. 在命令行中输入“ls -l install.log”——单独查看当前目录下名为install.log文件详细信息；
  5. 在命令行输入“ls /”——指定查看根目录下所有文件及目录；
  6. 在命令行输入“ls /root/.bash_profile”——显示root目录下名为.bash_profile的文件，虽然文件为隐藏形式也可显示出来，而直使用“ls /root”命令是看不到隐藏文件的。
  7. 在命令行中输入“ls /root/in*”——显示root目录下以字母in开头的所有文件。

    清屏命令：“clear”——清屏操作。

    更改目录命令（cd）：

  8. “cd /”——返回到根目录；
  9. “cd /home”——切换到home目录；
  10. “cd ..”——返回上一级目录；
  11. “cd /var/ftp/pub”——一次性切换到pub目录下；
  12. “cd /root”——切换到root用户目录下；
  13. TAB键自动补齐目录；两次TAB键显示冲突的目录。

查看当前路径命令pwd：“pwd”——显示当前目录所在绝对路径；

创建目录命令mkdir：

  1. “mkdir a”——在当前目录下创建一个名为a的新目录；
  2. “mkdir a b c”——在当前目录下同时创建多个目录；
  3. “mkdir /123”——在指定目录下创建新目录123；
  4. “mkdir -p /a/b/c”——在根目录下创建出三个嵌套的目录a/b/c;

创建文件命令（touch）：

  1. “touch 123”——新建名为123的文件；
  2. “touch a b c”——同时新建多个文件；

删除命令（rm）：

  1. “rm 123”+“y”——删除文件123（y——确认删除）；
  2. “rm -f”——强行删除文件123，不需要确认；
  3. “rm -r abc”——删除目录abc（“y”——确认删除）；
  4. “rm -fr abc”——强行删除目录abc，不需要确认；
  5. “rm -f *a”——强行删除以a开头的文件，不需要确认。

拷贝命令（cp）：

  1. “cp 123 /var/ftp/pub”——把名为123的文件拷贝到/var/ftp/pub目录下；
  2. “cp 123 /var/ftp/pub/456”——拷贝文件123到pub目录下并更名为456；
  3. “cp -r /var/ftp/pub /home”——把pub目录拷贝到home。

移动命令（mv）：

  1. “mv 123 /var/ftp/pub”——把名为123的文件移动到/var/ftp/pub目录下；
  2. “mv 123 /var/ftp/pub/456”——移动文件123到pub目录下并更名为456；
  3. “mv /var/ftp/pub /home”——把pub目录移动到home下；
  4. “mv 123 456”当前目录下文件123被更名为456。

更改权限命令（chmod）：

  1. “ls -l abc”——查看它的权限；（-建立者拥有的权限-组内其他成员权限-其他用户的权限-）；
  2. “chmod 777 abc”——对文件abc的权限进行更改（7可读可写可执行，77是最高权限：所有用户可读可写可执行）；
  3. 4——只读；2——可写；1——可执行； 例如：4+2=6 6代表可读可写。

查看文本内容命令（cat/less）：

  1. “cat /etc/passwd”——可查看文本文件passwd的内容，但只能查看最后一页；
  2. “less /etc/passwd”——可使用上下光标键及上下翻页键查看文本文件passwd的全部内容，查看完毕后q键退出。

解压命令（tar）：

  1. “tar -zxvf abc.tar.gz”解压abc.tar.gz；
  2. “tar -jxvf abc.tar.bz2”解压abc.tar.bz2。

三. 在终端命令行中启动vi，并进行文本的添、删、查、改以及保存、退出等基本操作。

创建文本文件：

  1. 假设当前目录下不存在名为abc的文件，则输入“vi abc”并回车，可在当前目录下新建一个名为abc的文本文件，进入vi编辑器后，按一次“I”键进入插入模式（屏幕底端显示INSERT）；
  2. 录入完成后按一次“Esc”键返回到命令状态，然后输入一个冒号，此时屏幕底行会出现一个“：”，在其后输入“wq”并回车，可对文件进行存盘退出，若不想保存，则输入“q！”并回车直接强行退个出。

打开文本文件：

“vi abc”——打开abc；

删除文本内容:

  1. 文件打开后，其内容显示在屏幕上，要对其中某一行进行删除，可在命令状态下（即非插入模式下），把光标定位到想要删除的行（可使用上下光标键），然后连续按两次“D”键即可删除该行；
  2. 要删除多行，可在按D键前加入要删除的行数，比如要连续删除7行，可把光标定位到要删除内容的首行，然后按“7DD”三个键即可；
  3. 要删除从光标位置开始至本行结尾内容，可在定位好光标后，顺序按“D$”两个键即可；
  4. 要删除从光标位置开始至本行开头的内容，可在定位好光标后，顺序按“D0”两键即可。

拷贝文本内容：

A.要拷贝某一行的内容，在命令状态下，把光标定位到想要拷贝的行，然后连续快速按两次“Y”键，随后把光标定位到想要黏贴的地方，然后按一次“P”键即可完成拷贝；
B.要进行多行拷贝，可在按YY前加入要拷贝的行数，比如要连续拷贝6行，可把光标定位到要拷贝的内容的首行，然后再按顺序按“6YY”，随后把光标定位到想要黏贴的地方，然后再按一次“P”键即可完成多行拷贝，粘贴可进行多次；

编辑文本内容：

文本打开后，按“I”键进入插入模式。

字符串查找：

A.打开文本文件，在底行模式（“：”）下输入“/SSN”并回车，则可找到文本中自光标处开始向下找到第一个SSN字符串，找到后光标会移动到该行；
B.要继续查找，按“N”即可；
C.要向上查找，把上面“/”换成“？”即可，如“？SSN”。

字符串替换：

A.在底行模式下输入“%s/var/abc”可把本行第一个var改成abc；
B.在底行模式下输入“%s/var/abc/g”可把本行所有var改成abc；
C.在底行模式下输入“1，20s/var/abc/g”可把1-20行中所有var替换为abc。

设置行号：

A.在底行模式下（在命令状态下输一个冒号），输入“set nu”并回车，可显示出行号，便于查找操作，行号本身并不会加到实际内容中；
B.要跳转到某行，直接在底行模式下输入行号并回车即可，要跳转到最后一行，输入“$”并回车，要跳转到第一行，输入“0”并回车即可；
C.要取消行号显示，可在底行模式下输入“set nonu”并回车即可。

在Linux系统下还可以使用vim，它的功能与vi一致，但在文本的显示上使用了颜色来加以区分，特别适合程序编辑。

Hugo搭建博客

虽然决定放弃Hugo了，但是搭建博客的方法还是放上来吧，万一哪天我又想用它了

安装Hugo

‘’’shell
$ brew install hugo
$ hugo version
‘’’

创建主页

‘’’shell
$ hugo new site myblog
$ cd myblog
‘’’

下载主题(https://themes.gohugo.io)

‘’’shell
$ cd themes
$ git submodule add https://github.com/alexandrevicenzi/soho.git
‘’’

修改配置

‘’’shell
$ cd .. #回到根目录myblog
$ vi config.toml
‘’’

创建文章

‘’’shell
$ hugo new post/blog.md （#在content文件下）
‘’’

编辑文章

‘’’shell
$ cd content/post
$ vi blod.md
‘’’

本地启动

‘’’shell
$ hugo server -t soho - -buildDrafts (在myBlog下)
‘’’

部署到远端

$ hugo - -theme=soho - -baseUrl=“https://charonwang77.github.io/" - -buildDrafts
$ cd public
$ git init
$ git add *
$ git commit -m “我的Hugo博客第1次提交”
$ git remote add origin https://github.com/charonwang77/charonwang77.github.io.git
$ git push -u origin master
